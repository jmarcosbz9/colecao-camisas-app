— Atualmente, após o clique que ativa o preenchimento no campo Preco, a box ativa está sangrando à direita com relação a mesma box quando inativa. Objetivo: igualá-las (alinha-lhas) mantendo R$ como hint inside box;
— Flexibilidade do preenchimento do campo Preco: caso o usuário digite um valor inteiro; exemplo: 3425 e teclar enter, tal valor deve ser completado com duas casas decimais no valor 00 e apresentado como R$ 3.425,00 a ele. Obs.: no banco de dados camisas → tabela outros_times o campo Preco é decimal(10,2);
— Atualmente, se o usuário clicar em mais de uma dropdown box de campos tipo enum e não selecionar algum valor para alguma delas, todos ficam abertas no formulário até o usuário selecionar/preencher algum campo com algum valor. Responsividade desejada: caso o usuário abra algum menu dropdown e não selecione valor algum, porém clicar fora do menu, o dropdown menu deverá ser fechado;
— Atualmente, em campos não enum tais como Equipe, Preco, Numero, caso o usuário preencha o conteúdo desejado e tecle enter para confirmar o preenchimento, todo o form fecha sem criar o registro. Comportamento desejado: ao teclar enter após preencher tais campos deve-se preservar o conteúdo e dar continuidade ao preenchimento dos outros campos;
— UX drag & drop + Windows Style WYSIWYG para seleção das fotos está funcional. Porém, ao carregar uma foto é apresentada o nome dela e, mesmo assim, abaixo da área drag & drop. Responsividade desejada: deve ser apresentada a thumb dela dentro da área drag & drop e abaixo da thumb, aí, sim, o nome do arquivo. As thumbs dever ir se alinhando horizontalmente no campo drag & drop à medida que o usuário vai "uploadando" fotos. Obs.: o mecanismo de envio → frontend → backend → armazenagem física das fotos, criação das suas thumbs → preenchimento JSON do campo fotos na tabela outros_times estão ok.
